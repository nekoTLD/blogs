Web + Logの苦悩。

Web + Log、つまりはブログ、をぼちぼち書いているのだが、これに関して悩むことがいろいろと多い。なので今回は、それについて。

Written by NA - 2022, Jan3 (2) - Article No: 2


リンクフリーは本当か？

Web + Logなのだから、やはりブログはWebの一部である。で、ここで頭を悩ませる存在が、リンクと引用である。もっとも私個人はWebはリンクフリーであると願っているが、残念ながらことはそう簡単に進まない。記事を書くうえで、引用というのは多々ある行為かと思うが、Webではそれは多くの場合リンクを伴うし、そうあるべきだと思う。(それがWebの良さであろう) しかしながら、リンクをするということは、被リンク側のWebサイトがリンクを許容していなければやりづらい。技術的には可能であろうが、ややこしい権利関係の話を避けようと思うと、どうしても明示的にリンクを許容しているWebサイト以外には、リンクというのはしづらいのである。特に、新聞社など、リンクに関して否定的なポリシーを掲げているところがどうも多いように感じるからなおさらだ。結果的にこのWebサイトではほとんどリンクはないし、引用は避けている。しかしそうすると、如何せん、記事が書きづらい。どうしたものだろうか。
もっとも、リンク程度のことで、これは杞憂なのかもしれないが、Webが世界に開かれた存在であることを考えると、やはり気にせざるを得ない。


Big G

Big G、そう、Googleのことだ。GoogleはWebのあらゆるところに存在する。それをありがたがる人もいれば、不気味に思う人もいる。私はどちらかといえば後者だ。しかしなぜGoogleがWebの至る所に存在するのか？それは便利だからだ。Google AnalyticsやAdSenseは勿論、Google Fontsなんかも有名どころだろう。AdSenseに関しては、私はそもそも追跡型Web広告に対して否定的なこともあり、このWebサイトに導入することはないが、AnalyticsやFontsなんかは割と悩んでいる。やはりアクセス解析はしたいし、あらゆる環境でほぼ同じレンダリングを実現しようとすると、Webフォントは魅力的だ。しかしながら、私自身はこれらがあまり好きではなく、如何に訪問者のプライバシーを確保しつつ、優れたWebサイトを作ればいいのか、という、葛藤に頭を悩ませている。
ここで、バナーを表示して、Opt In形式にする、ということも考えたが、Cookieバナーがいかにインターネットユーザから忌み嫌われているかを考えれば、その答えはNoである。また、Opt Out形式も当然Noだ。
